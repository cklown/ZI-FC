FormComplete Utility Script Demo - 1

Overview

This utility script builds off the existing logic. It provides an easy to deploy solution which addresses UI complications where customers lack developer resources.

  • Eliminates flicker; flicker briefly shows mapped fields before being hidden.
  • If form contains unmapped fields, jitter on load is potential trade off .
  • Jitter can be eliminated by applying a single CSS class 'zi_alwaysShow' directly on all unmapped fields.
  • Event generated forms created after 'Form Load Timeout' can now utilize FormComplete without any additional effort from customers.
  • Field containers can now be hidden, resolving potentially akward spacing issues between fields and or visible labels / other field related elements.

Implementation
Idealy, if the resource can be hosted by ZoomInfo, implementation could be as simple as providing the customer with the following code to place in their site's header.
There are five configuration values:

  • formIframe: string, query selector - If form loads in an iframe.
  • eventForm: boolean - Does the form not exist on page load?
  • formContainer: string, query selector - If eventForm is true, specify where the form will be created. This should be the closest parent that is available on load.
  • formSelector: string, query selector - Specify the form selector. Forms lacking a unique ID/Name/Class can also now use: form[action="MyFormActionURL"]
  • fieldContainer: string, query selector - Optional; If fields are reside within a container, specify the element's ID, Name, Class, or other attribute. If not provided, the field itself will be hidden.

Javascript
<script> window.ZIConfigurations = { formIframe: 'iframe[name="MyIframeName"]', eventForm: true, formContainer: 'div#MyFormContainerID', formSelector: 'form[action="MyFormActionURL"]', fieldContainer: 'div.MyFeildContainerClass' } window.ZIProjectKey = "12345678901234567890"; // Do not modify below. var zi = document.createElement('script'); (zi.type = 'text/javascript'), (zi.async = true), (zi.src = 'https://js.zi-scripts.com/zi-fc-af.js'), document.head.appendChild(zi); </script>

About

This example contains a form with:

  • 5 mapped fields (email, name, company, phone, job title)
  • A text and textarea field (Product Interests and Message)

This demo shows Antiflicker functionality and the potential jitter that can occur.
The purpose: hide mapped fields when form is loaded. If the form contains any unmapped fields, the client may add the custom class 'zi_alwaysShow' to each field to prevent jitter.
